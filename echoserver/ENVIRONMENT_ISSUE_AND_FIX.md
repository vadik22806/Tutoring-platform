# Проблема окружения и её решение

## Контекст

Проект `echoserver` работает на старом стеке зависимостей с MongoDB через `djongo`.
В рабочем `requirements.txt` используются версии:

- `Django==3.1.12`
- `djongo==1.3.7`
- `djangorestframework==3.12.4`
- `djangorestframework_simplejwt==4.8.0`
- `sqlparse==0.2.4`

## Проблема

При запуске сервера на Python `3.12` в виртуальном окружении `.venv` появлялась ошибка:

- `ModuleNotFoundError: No module named 'pkg_resources'`
- вторая ошибка: `ModuleNotFoundError: No module named 'distutils'`

Это произошло не из-за кода самого сайта, а из-за несовместимости пакетов и Python 3.12.

## Почему это произошло

1. `rest_framework_simplejwt` импортируется вместе с DRF и требует `pkg_resources`.
2. В текущем окружении `setuptools` не поставлял корректно `pkg_resources` на Python 3.12.
3. Django 3.1 пытается импортировать `distutils`, а в Python 3.12 эта часть экосистемы изменилась.
4. Переход на API/DRF просто активировал этот путь импорта, поэтому проблема стала заметной.

## Решение

### Вариант 1: вернуть рабочее состояние текущего стека

1. Оставить зависимости из `requirements.txt`.
2. Восстановить виртуальное окружение и установить:
   - `Django==3.1.12`
   - `djangorestframework==3.12.4`
   - `djangorestframework_simplejwt==4.8.0`
   - `djongo==1.3.7`
   - `sqlparse==0.2.4`
   - `setuptools==82.0.0`
3. Запустить `python echoserver/manage.py check`.

В результате проект снова становится рабочим, но остаётся ограничение на Python версии.

### Вариант 2: перейти на совместимый стек под Python 3.12

Для поддержки Python 3.12 необходимо обновить стек:

- `Django` >= 4.2
- `djangorestframework` >= 3.14
- `djangorestframework_simplejwt` >= 5.x
- `sqlparse` >= 0.5.0
- отказаться от `djongo` или перейти на другой MongoDB адаптер

Это более долгий путь, но он снимет ограничение по версии Python и позволит использовать актуальные зависимости.

## Рекомендация

Для текущего проекта на данном этапе лучше:

- использовать Python 3.10/3.11 для этого репозитория
- вернуть рабочие зависимости из `requirements.txt`
- продолжить разработку API на стабильном стеке

Если нужен переход на Python 3.12, то требуется рефакторинг миграции с `djongo` на современный MongoDB-адаптер и обновление Django/DRF.
